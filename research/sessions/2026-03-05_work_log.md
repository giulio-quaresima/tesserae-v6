## Fix 3 (research only): Relax syntax channel lemma gate

### Current architecture
- Pruning gate: only pairs sharing >=1 lemma are candidates
- Scoring function: scores based on deprel/upos agreement at shared lemma positions. Returns 0.0 with zero shared lemmas.
- Both barriers must be addressed for the Thomas tricolon case.

### Cost estimate
- Georgics × DRN 6: 2,509,479 all-pairs vs 271,271 lemma-gated (9.3x increase)
- Lucan BC 1 × Aeneid: 1,367,180 all-pairs vs 109,651 gated (12.5x increase)
- At current speeds: ungated would add ~4+ minutes per search

### Proposed approaches
1. **Pure structural similarity:** Compare head arrays and deprel sequences directly (not via shared lemmas). Two-tier: fast fingerprint filter → detailed comparison. Risk: many false positives from common Latin verse patterns.
2. **Semantic-gated syntax:** If two lines have SPhilBERTa cosine > 0.5 but no shared lemmas, also check syntax. Leverages existing data, much cheaper than all-pairs.
3. **Hybrid:** Combine semantic gate with structural fingerprint pre-filter.

Detailed notes in `research/sessions/2026-03-05_georgics_lucretius_diagnosis.md`.

## Fix 3: Structural Fingerprint Matching (Option C) — IMPLEMENTED

Added a second path to `find_syntax_matches()` in `backend/fusion.py` that matches lines by dependency head pattern fingerprints without requiring shared lemmas.

### How it works
1. **Path A (existing):** Lemma-gated candidates → scored by `_compute_syntax_score()` (deprel/upos at shared lemma positions)
2. **Path B (new):** Head-pattern fingerprint index → only exact fingerprint matches → scored by `_compute_structural_score()` (head match 0.5 + deprel agreement 0.3 + upos agreement 0.2)

### Key characteristics
- Fingerprints are highly selective: 261K unique patterns from 542K corpus lines
- Very few novel candidates per search (590–1,508), negligible performance impact
- Only fires for pairs with identical non-punctuation head arrays of length >= 3

### Benchmark result: SAFE
Full 5-benchmark: **800/862 (92.8%)** — identical to baseline. Zero regression.

## Fix 5: Two-tier semantic recovery for structural pairs

After all line channels run, look up cosine similarity for structural fingerprint pairs that were filtered by the semantic channel's top-100 cap. Inject qualifying results (cosine >= 0.5) into semantic channel before fusing.

### How it works
1. `_recover_semantic_for_structural()` loads precomputed embeddings
2. For each structural pair not already in semantic results, computes cosine similarity
3. Pairs above 0.5 threshold injected as semantic results
4. Fusion naturally combines syntax_structural + semantic evidence

### Impact
- Georg 3.481/DRN 6.1140: rank 402 → **rank 245** (2 channels: syntax_structural + semantic)
- Score: 1.954 (syntax_structural 1.5 × 0.84 + semantic 1.2 × 0.579 + convergence)
- Benchmark: 800/862 (92.8%), zero regression
- Typical recovery: 302 of 697 structural pairs get semantic scores (Georgics × DRN 6)

### New function
- `_recover_semantic_for_structural()` in `backend/fusion.py`
- Called in both `run_fusion_search()` and `iter_fusion_search()`
